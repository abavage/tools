# create the policy
aws iam create-policy --policy-name instance-s3readonly-policy --policy-document file://instance-policy.json --query 'Policy.Arn'

# attach the policy to the role -- this is attached to the role s3readonlyrole
aws iam attach-role-policy --policy-arn arn:aws:iam::559050246145:policy/instance-s3readonly-policy --role-name instance-s3readonly-role




# flow
1) Use RoleA trustpolicy to become RoleA
2) RoleA uses it's permission policy to request access to RoleB
3) RoleB uses it's trust policy to veriy it knows and trusts RoleA
4) STS issues temporary creds for RoleB
5) The entity (application/user) used RoleB permissions policy to read from S3

# steps
1) assume roleA as an entity (user or application)
2) set the env vars 
3) check that it assumed the correct role
4) assume roleB AS roleA
5) set the env vars
6) check that it assumed the correct role
7) list the buckets

# files
role-a-trust.json - what/who can use this role
role-a-policy.json - what can this role do eg assume a role
role-b-trust.json - what/who can use this role
role-b-policy.json - what can this role do eg list buckets


# Create roleA with trust
